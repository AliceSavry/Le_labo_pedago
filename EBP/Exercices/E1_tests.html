Exercice 1 - TESTS

Objectifs de l'exercice

Cet exercice vous permettra de vous familiariser avec les fonctionnalités de base d'EBP Gestion Commerciale à travers une série de manipulations simples.

  • Création et paramétrage d'un dossier
  • Gestion des articles de base
  • Création de documents commerciaux simples
  • Manipulation des fonctions essentielles

Mise en situation

Vous êtes en charge de tester les différentes fonctionnalités d'EBP Gestion Commerciale pour une nouvelle installation. Vous devez créer un dossier de test et effectuer une série d'opérations pour vérifier le bon fonctionnement du logiciel.

Étape 1 : Création du dossier

  1. Créer un nouveau dossier
    • Nom : TESTS
    • Exercice : 01/01/2024 au 31/12/2024
    • Devise : Euro
  2. Paramétrer les informations société
    • Raison sociale : Société Test
    • Adresse : 1 rue des Tests
    • Code postal : 75001
    • Ville : Paris

Étape 2 : Création des articles

  1. Créer 3 articles de test
    • TEST001 - Article 1 (Prix HT : 100€)
    • TEST002 - Article 2 (Prix HT : 200€)
    • TEST003 - Article 3 (Prix HT : 300€)
  2. Pour chaque article :
    • Définir un stock initial de 10 unités
    • Appliquer une TVA de 20%
    • Ajouter une description détaillée

Étape 3 : Création d'un client test

  1. Créer un client
    • Code : CLI001
    • Raison sociale : Client Test
    • Catégorie tarifaire : Standard
  2. Paramétrer les conditions de paiement
    • Mode de règlement : Chèque
    • Délai de paiement : 30 jours

Étape 4 : Documents commerciaux

  1. Créer un devis
    • Inclure les 3 articles créés
    • Appliquer une remise de 5%
  2. Transformer le devis en commande
  3. Créer un bon de livraison
  4. Générer la facture

Étape 5 : Vérifications

  • Contrôler les documents
    • Vérifier la cohérence des prix
    • Contrôler les calculs de TVA
    • Vérifier les totaux
  • Vérifier les stocks
    • État des stocks après livraison
    • Mouvements de stock
  • Contrôler les statistiques
    • Chiffre d'affaires
    • Marge

Points d'attention

  • Noter toute anomalie rencontrée
  • Vérifier la cohérence des données
  • Tester les impressions de documents
  • Valider les calculs automatiques
  • Contrôler les reports en comptabilité

Validation de l'exercice

L'exercice sera considéré comme réussi lorsque :

  • Tous les documents sont correctement créés
  • Les calculs sont exacts
  • Les stocks sont correctement mis à jour
  • Les statistiques sont cohérentes
  • Les documents peuvent être imprimés
